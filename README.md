# AKD Image

AKD Image – статический веб-сервис с набором инструментов для обработки изображений прямо в браузере.

Файлы не загружаются на сервер AKD Image: браузер читает их через `FileReader` или локальные object URL, обрабатывает через Canvas API и сохраняет обратно как `Blob`. Основные зависимости и шрифты находятся в репозитории.

## Возможности

- Сжатие JPG, PNG и WebP.
- Изменение размера по пикселям, процентам или длинной стороне.
- Добавление текстового водяного знака или логотипа.
- Обрезка изображения по области и точным координатам.
- Конвертация между JPG, PNG и WebP.
- Поворот и отражение изображений.
- Фотоэффекты: яркость, контраст, насыщенность, размытие, сепия, grayscale, инверсия.
- Генератор мемов.
- Разделение изображения на части или сетку с ZIP-скачиванием.
- Скругление углов с PNG-результатом.
- Пикселизация.
- Удаление EXIF/метаданных через пересохранение изображения.
- Преобразование изображения в Data URL или Base64 и обратно.
- Создание `favicon.ico` и набора PNG-иконок с ZIP-скачиванием.
- Извлечение цветовой палитры с HEX, RGB, долей цвета и экспортом в PNG.
- Размытие выбранной области для скрытия лиц, номеров, логинов и других личных данных.
- Создание PDF из одного или нескольких изображений.
- Создание коллажа из нескольких изображений.
- Пометки на изображениях: текст, кисть, стрелки, круги, овалы, рамки и перемещение элементов.

## Возможности интерфейса

- Светлая и темная тема интерфейса с сохранением выбора в браузере.

Пакетные инструменты позволяют скачивать результаты по отдельности или одним ZIP-архивом.

## Приватность

Файлы остаются на устройстве пользователя. В обычных сценариях инструменты не отправляют изображения на сервер.

Исключение: в инструменте водяного знака пользователь может вручную вставить ссылку на внешний логотип. В этом случае браузер загружает указанное изображение по этой ссылке.

## Технологии

| Технология | Назначение |
|---|---|
| HTML, CSS, JavaScript | Основа интерфейса |
| Canvas API | Обработка и пересохранение изображений |
| FileReader API и object URL | Чтение выбранных файлов |
| Blob и URL.createObjectURL | Скачивание результатов |
| Clipboard API | Копирование результата в буфер в отдельных инструментах |
| JSZip | Создание ZIP-архивов |
| jsPDF | Создание PDF-файлов |
| Inter | Локальный UI-шрифт |

Сборщик, npm и сервер приложения не требуются.

## Запуск и размещение

Проект можно открыть напрямую через `index.html` или запустить любым локальным статическим HTTP-сервером.

Для размещения на Vercel в репозитории находится `vercel.json` с Content Security Policy и другими защитными HTTP-заголовками.

## Общие модули

`js/layout.js` добавляет общий header, footer и контейнер для toast-уведомлений.

`js/core.js` содержит общие утилиты:

- `Toast` – уведомления.
- `UIUtils` – общие UI-помощники.
- `FileUtils` – чтение, валидация, сохранение, ZIP и Canvas helpers.
- `Dropzone` – drag-and-drop и выбор файлов.
- `FileListManager` – список файлов, статусы, progress, кнопки скачивания.

## Форматы

Основные инструменты принимают JPG, PNG и WebP.

При экспорте:

- JPG получает белую подложку, потому что формат не поддерживает прозрачность.
- PNG и WebP сохраняют прозрачность, если она есть.
- Скругление углов всегда сохраняет PNG.
- Base64-конвертер работает с JPG, PNG и WebP.
- Генератор favicon создаёт PNG разных размеров и контейнер `favicon.ico`.
- Генератор палитры экспортирует результат в PNG.
- Размытие области сохраняет результат в JPG, PNG или WebP.
- Инструмент изображений в PDF принимает JPG, PNG и WebP и сохраняет результат в PDF.
- Коллаж принимает JPG, PNG и WebP и сохраняет результат в JPG, PNG или WebP.
- Пометки на изображениях принимают JPG, PNG и WebP и сохраняют результат в JPG, PNG или WebP.
- ZIP создается локально через `js/vendor/jszip.min.js`.

## Ограничения ресурсов

Для защиты вкладки от чрезмерного расхода памяти действуют централизованные ограничения:

- до 50 МБ на один файл;
- до 50 файлов и 250 МБ за одну загрузку;
- до 40 мегапикселей и 16 384 px по стороне;
- до 500 МБ исходных данных при создании ZIP.
