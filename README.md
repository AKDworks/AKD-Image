# AKD Image

AKD Image — статический веб-сервис с инструментами для обработки изображений прямо в браузере.

Проект не использует серверную загрузку файлов: изображения читаются через `FileReader`, обрабатываются через Canvas API и скачиваются обратно как `Blob`. Основные зависимости лежат локально в репозитории.

## Возможности

- Сжатие JPG, PNG и WebP.
- Изменение размера по пикселям, процентам или длинной стороне.
- Добавление текстового водяного знака или логотипа.
- Обрезка изображения по области и точным координатам.
- Конвертация между JPG, PNG и WebP.
- Поворот и отражение изображений.
- Фотоэффекты: яркость, контраст, насыщенность, размытие, сепия, grayscale, инверсия.
- Генератор мемов.
- Разделение изображения на части или сетку с ZIP-скачиванием.
- Скругление углов с PNG-результатом.
- Пикселизация.
- Удаление EXIF/метаданных через пересохранение изображения.

## Приватность

Файлы остаются на устройстве пользователя. В обычных сценариях инструменты не отправляют изображения на сервер.

Исключение: в инструменте водяного знака пользователь может вручную вставить ссылку на внешний логотип. В этом случае браузер загружает указанное изображение по этой ссылке.

## Технологии

| Технология | Назначение |
|---|---|
| HTML, CSS, JavaScript | Основа интерфейса |
| Canvas API | Обработка и пересохранение изображений |
| FileReader API | Чтение выбранных файлов |
| Blob и URL.createObjectURL | Скачивание результатов |
| Clipboard API | Копирование результата в буфер в отдельных инструментах |
| JSZip | Создание ZIP-архивов |
| Inter | Локальный UI-шрифт |

Сборщик, npm и сервер приложения не требуются.

## Запуск

Проект можно открыть как обычный статический сайт.

Самый простой вариант:

```text
index.html
```

Для более надежной проверки лучше запустить любой локальный статический сервер из корня проекта, например через Node.js или расширение редактора. Отдельная сборка не нужна.

## Общие модули

`js/layout.js` добавляет общий header, footer и контейнер для toast-уведомлений.

`js/core.js` содержит общие утилиты:

- `Toast` — уведомления.
- `Modal` — базовое модальное окно.
- `UIUtils` — общие UI-помощники.
- `FileUtils` — чтение, сохранение, ZIP, Canvas helpers.
- `Dropzone` — drag-and-drop и выбор файлов.
- `FileListManager` — список файлов, статусы, progress, кнопки скачивания.

## Форматы

Основные инструменты принимают JPG, PNG и WebP.

При экспорте:

- JPG получает белую подложку, потому что формат не поддерживает прозрачность.
- PNG и WebP сохраняют прозрачность, если она есть.
- Скругление углов всегда сохраняет PNG.
- ZIP создается локально через `js/vendor/jszip.min.js`.

## Добавление нового инструмента

Этот раздел предназначен для владельца проекта и внутренней разработки AKD Image.

1. Создать страницу в `pages/`.
2. Подключить `../css/main.css`, `../js/layout.js` и `../js/core.js`.
3. Использовать готовые помощники `Dropzone`, `FileListManager`, `FileUtils`, `Toast`.
4. Добавить карточку инструмента в `index.html`.
5. При необходимости добавить ссылку в массив `tools` в `js/layout.js`.
6. Добавить `title` и `meta description`.

Минимальная основа страницы:

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Новый инструмент — AKD Image</title>
  <meta name="description" content="Короткое описание нового инструмента.">
  <link rel="stylesheet" href="../css/main.css">
</head>
<body>
<script src="../js/layout.js?v=5"></script>

<main>
  <div class="container container--narrow tool-page">
    <div class="tool-page__header">
      <h1 class="tool-page__title">Новый инструмент</h1>
      <p class="tool-page__subtitle">Короткое пояснение для пользователя.</p>
    </div>
  </div>
</main>

<script src="../js/core.js"></script>
<script>
(function () {
  // Логика инструмента.
})();
</script>
</body>
</html>
```

## Проверка перед коммитом

Этот раздел также относится к внутренней разработке проекта.

Рекомендуемый минимум:

```powershell
node --check js\core.js
node --check js\layout.js
```

Также стоит открыть сайт локально и вручную проверить основной сценарий инструмента:

```text
выбрать файл -> обработать -> скачать результат
```

Для пакетных инструментов дополнительно проверить:

```text
несколько файлов -> удалить один файл -> обработать -> скачать ZIP
```

## Примечания для разработки

- Соблюдать условия из `LICENSE`.
- Не подключать внешние CDN без необходимости.
- Не добавлять серверную загрузку файлов для базовых инструментов.
- Vendor-файлы в `js/vendor/` не редактировать вручную.
- При работе с JPG учитывать отсутствие прозрачности.
- Для PNG и WebP сохранять прозрачность там, где это возможно.
- README должен описывать текущее состояние проекта, а не будущие планы.
